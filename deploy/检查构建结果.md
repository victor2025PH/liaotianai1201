# 检查构建结果

请在服务器上手动执行以下命令来诊断问题：

## 1. 检查文件是否存在

```bash
cd /home/ubuntu/liaotian/saas-demo
ls -la src/app/group-ai/accounts/page.tsx
wc -l src/app/group-ai/accounts/page.tsx
```

## 2. 检查构建后的路由文件

```bash
cd /home/ubuntu/liaotian/saas-demo
find .next -path '*group-ai/accounts*' -type f
find .next -name '*accounts*' -type f
ls -la .next/app/group-ai/accounts/ 2>/dev/null || echo "目录不存在"
```

## 3. 检查构建日志

```bash
cd /home/ubuntu/liaotian/saas-demo
# 如果有构建日志
cat /tmp/build.log | grep -i "group-ai\|accounts\|route" | tail -20
```

## 4. 检查Next.js配置

```bash
cd /home/ubuntu/liaotian/saas-demo
cat next.config.ts
```

## 5. 重新构建并查看详细输出

```bash
cd /home/ubuntu/liaotian/saas-demo
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && . "$NVM_DIR/nvm.sh"
nvm use 20
npm run build 2>&1 | grep -E "(Route|group-ai|accounts|Compiled|error)" | tail -50
```

## 6. 检查服务日志

```bash
sudo journalctl -u liaotian-frontend -n 50 --no-pager | grep -i "error\|404\|route"
```

## 可能的问题

1. **文件路径问题**：确保文件在 `src/app/group-ai/accounts/page.tsx`
2. **构建时路由未被包含**：检查构建输出中是否有该路由
3. **Next.js配置问题**：检查 `next.config.ts` 是否有路由相关的配置
4. **缓存问题**：清理 `.next` 和 `node_modules/.cache` 后重新构建
