# 账户中心访问指南

## ✅ 正确的访问地址

**前端服务地址**: http://localhost:3000

**账户中心页面**: http://localhost:3000/accounts

## 📍 页面结构

### 路由配置

- **路径**: `/accounts`
- **组件**: `Accounts` 组件
- **位置**: `admin-frontend/src/pages/Accounts.tsx`

### 标签页结构

账户中心页面包含3个标签页：

1. **账户列表** (`key: "list"`)
   - 显示所有账户的列表
   - 包含手机号、昵称、角色、状态等信息

2. **Telegram注册** (`key: "register"`)
   - 新开发的注册功能
   - 组件: `TelegramRegister`
   - 包含完整的注册流程

3. **生成Session** (`key: "generate"`)
   - 原有的Session生成功能
   - 组件: `GenerateSession`

## 🚀 访问步骤

### 1. 启动服务

```bash
# 启动后端（如果未启动）
cd admin-backend
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000

# 启动前端（如果未启动）
cd admin-frontend
npm run dev
```

### 2. 访问系统

1. 打开浏览器
2. 访问: **http://localhost:3000**
3. 登录系统（需要有效的账号密码）

### 3. 进入账户中心

**方法1**: 通过菜单
- 点击左侧菜单"账户中心"
- 自动跳转到 `/accounts`

**方法2**: 直接访问URL
- 访问: **http://localhost:3000/accounts**

### 4. 使用Telegram注册功能

1. 在账户中心页面
2. 点击"Telegram注册"标签页
3. 开始使用注册功能

## ⚠️ 重要提示

## 📝 常见问题

### Q: 访问5173端口可以吗？

A: 不建议。前端服务已配置为3000端口，应该访问 http://localhost:3000

### Q: 如何确认前端服务在哪个端口？

A: 运行以下命令：
```bash
netstat -ano | findstr :3000
```

### Q: 账户中心页面显示错误？

A: 检查：
1. 前端服务是否在3000端口运行
2. 是否已登录系统
3. 浏览器控制台是否有错误

## ✅ 配置确认

### 前端端口配置

**文件**: `admin-frontend/vite.config.ts`
```typescript
server: {
  port: 3000,
  strictPort: true
}
```

### 后端CORS配置

**文件**: `admin-backend/app/core/config.py`
```python
cors_origins: str = "http://localhost:3000,http://localhost:5173,http://localhost:8080"
```

✅ 后端已配置允许3000端口访问
