# 对接方案选择指南

> **创建日期**: 2025-11-15

---

## 快速选择

### 场景1：两个系统在同一服务器/网络 ✅ 推荐

**选择**：数据库对接

**配置**：
```bash
GAME_DATABASE_URL=sqlite:///./path/to/game_system/data.sqlite
```

**优点**：
- ✅ 游戏系统无需改动
- ✅ 实时、准确
- ✅ 性能好

---

### 场景2：两个系统在不同服务器/网络

**选择1**：Telegram 按钮检测（默认）

**配置**：无需配置

**优点**：
- ✅ 无需任何配置
- ✅ 自动检测

**选择2**：HTTP API 对接

**配置**：
```bash
GROUP_AI_GAME_API_BASE_URL=http://game-system-api:8000
GROUP_AI_GAME_API_KEY=your_api_key
```

**要求**：游戏系统需要提供 API

---

## 详细对比

| 对接方式 | 游戏系统改动 | 配置复杂度 | 性能 | 推荐度 |
|---------|------------|-----------|------|--------|
| 数据库对接 | ❌ 无需 | ⭐⭐ 简单 | ⭐⭐⭐⭐⭐ 优秀 | ⭐⭐⭐⭐⭐ 最推荐 |
| Telegram 按钮检测 | ❌ 无需 | ⭐ 无需 | ⭐⭐⭐⭐ 良好 | ⭐⭐⭐⭐ 推荐 |
| 直接调用函数 | ❌ 无需 | ⭐⭐ 简单 | ⭐⭐⭐⭐⭐ 优秀 | ⭐⭐⭐⭐ 推荐 |
| HTTP API | ✅ 需要 | ⭐⭐⭐ 中等 | ⭐⭐⭐ 一般 | ⭐⭐⭐ 可选 |

---

## 实施建议

### 第一步：尝试数据库对接

1. 获取游戏系统数据库连接字符串
2. 配置 `GAME_DATABASE_URL`
3. 测试查询功能

### 第二步：如果数据库不可用，使用 Telegram 按钮检测

1. 无需配置
2. 自动检测按钮
3. 测试参与功能

### 第三步：如果需要更高级功能，考虑 HTTP API

1. 与游戏系统协商 API 接口
2. 实现 API 接口
3. 配置 API 连接

---

**推荐**：优先使用数据库对接，简单高效，无需游戏系统改动！
