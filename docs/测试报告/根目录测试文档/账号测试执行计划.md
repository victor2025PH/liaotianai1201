# 账号测试执行计划

## 目标
1. 扫描所有session文件（8个）
2. 批量创建账号（会自动分配到服务器）
3. 测试启动每个账号
4. 记录可用/不可用账号及原因
5. 对于可用的账号，更新数据库
6. 对于不可用的账号，生成通知

## 执行步骤

### 步骤1：批量创建账号
- 在"添加账号"对话框中
- 点击"選擇 Session"按钮
- 选择所有8个session文件
- 选择剧本：000新人欢迎剧本
- 批量创建账号

### 步骤2：测试启动账号
- 对每个账号点击"啟動"按钮
- 记录启动结果：
  - ✅ 成功：账号可用
  - ❌ 失败：记录失败原因

### 步骤3：分类账号
- **可用账号**：能够成功启动的账号
- **不可用账号**：启动失败的账号

### 步骤4：处理结果
- 对于可用的账号：确认已正确分配到服务器
- 对于不可用的账号：记录失败原因并生成通知

## 预期结果

### 可用账号
- 状态：online 或 running
- 服务器：已分配
- 可以创建群组和启动群聊

### 不可用账号
- 状态：error 或 offline
- 失败原因：
  - Session文件无效
  - Telegram连接失败
  - 需要重新验证
  - 其他错误

## 通知格式

对于不可用的账号，将生成通知文件，包含：
- 账号ID
- Session文件名
- 失败原因
- 建议处理方式
