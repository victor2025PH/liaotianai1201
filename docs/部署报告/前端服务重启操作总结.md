### 方案1：等待服务完全启动（推荐）

Next.js 开发服务器首次启动需要：

2. 启动开发服务器
3. 建立热重载连接

**预计时间**: 30-60秒

**操作**:
1. 等待 30-60 秒
2. 访问 http://localhost:3000
3. 检查浏览器控制台是否有错误

### 方案2：检查服务日志

查看前端服务的实际输出：
```bash
cd saas-demo
npm run dev
```

查看是否有编译错误或启动错误。

### 方案3：检查端口占用

确认端口 3000 是否被占用：
```bash
netstat -ano | findstr ":3000"
```

如果被占用，可以：
1. 终止占用端口的进程
2. 或使用其他端口（修改 package.json 中的端口配置）
