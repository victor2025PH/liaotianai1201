## 项目概述

### 1.1 项目简介

Smart TG Business AI 是一个**专业级 Telegram 群组 AI 账号智能管理系统**，支持：

- **多账号管理**：批量管理 1-1000+ 个 Telegram AI 账号
- **剧本驱动对话**：基于 YAML 配置的智能对话剧本系统
- **自动化任务**：自动聊天、红包参与、角色分配等
- **实时监控**：账号状态、对话质量、系统性能监控
- **权限管理**：基于角色的访问控制（RBAC）
- **通知系统**：实时通知推送（WebSocket + 邮件 + Webhook）

### 1.2 核心价值

- ✅ **规模化运营**：支持大规模 AI 账号矩阵运营
- ✅ **智能化互动**：基于剧本和 AI 的自然对话
- ✅ **自动化参与**：红包游戏等群组活动自动参与
- ✅ **可观测性**：完整的监控和调优能力
- ✅ **多语言支持**：简体中文、繁体中文、英文

## 使用指南

### 4.1 快速开始

#### 4.1.1 登录系统

1. 访问系统首页
2. 使用默认账号登录：
   - 邮箱: `admin@example.com`
   - 密码: `changeme123`
3. **重要**: 首次登录后请立即修改密码

#### 4.1.2 首次使用流程

**步骤 1: 创建剧本**
1. 进入「群组AI」→「剧本管理」
2. 点击「创建剧本」
3. 填写剧本信息（名称、描述）
4. 配置场景和触发条件
5. 保存并提交审核

**步骤 2: 导入账号**
1. 进入「群组AI」→「账号管理」
2. 点击「扫描 Session」或「上传 Session」
3. 选择 Session 文件
4. 系统自动识别并导入账号

**步骤 3: 分配角色**
1. 进入「群组AI」→「角色分配」
2. 选择剧本 → 点击「提取角色」
3. 选择账号 → 分配角色
4. 点击「应用分配」

**步骤 4: 创建自动化任务**
1. 进入「群组AI」→「自动化任务」
2. 点击「创建任务」
3. 选择任务类型和参数
4. 设置调度规则
5. 启动任务

### 4.2 核心功能使用

#### 4.2.1 剧本管理

**创建剧本**:
1. 点击「创建剧本」按钮
2. 填写基本信息：
   - 名称：剧本名称
   - 描述：剧本用途说明
   - 标签：便于分类
3. 配置场景：
   - 添加场景
   - 设置触发条件（关键词、时间、事件等）
   - 配置回复内容（文本、AI生成、变量替换）
4. 保存并提交审核

**剧本审核**:
- 剧本创建后需要审核才能使用
- 管理员可以在「剧本管理」页面审核剧本
- 审核通过后剧本可用于角色分配

#### 4.2.2 账号管理

**导入账号**:
- **方式 1**: 扫描 Session 文件
  - 点击「扫描 Session」
  - 选择包含 Session 文件的目录
  - 系统自动扫描并导入
- **方式 2**: 上传单个 Session 文件
  - 点击「上传 Session」
  - 选择 Session 文件
  - 填写账号信息（可选）

**启动账号**:
1. 在账号列表中选择账号
2. 点击「启动」按钮
3. 系统自动连接 Telegram
4. 账号状态变为「在线」

**停止账号**:
1. 选择运行中的账号
2. 点击「停止」按钮
3. 账号断开连接，状态变为「离线」

#### 4.2.3 自动化任务

**创建任务**:
1. 点击「创建任务」
2. 选择任务类型：
   - **自动聊天**: 基于剧本自动回复
   - **红包参与**: 自动检测并参与红包
   - **群组管理**: 自动管理群组
3. 配置任务参数：
   - 选择账号/群组
   - 设置执行规则
   - 配置触发条件
4. 设置调度：
   - 立即执行
   - 定时执行
   - 循环执行
5. 启动任务

**监控任务**:
- 在任务列表中查看任务状态
- 点击任务查看执行日志
- 可以暂停/恢复/停止任务

#### 4.2.4 权限管理

**创建用户**:
1. 进入「权限管理」→「用户管理」
2. 点击「创建用户」
3. 填写用户信息：
   - 邮箱（作为用户名）
   - 密码
   - 姓名
4. 分配角色
5. 保存

**创建角色**:
1. 进入「权限管理」→「角色管理」
2. 点击「创建角色」
3. 填写角色信息
4. 分配权限（选择权限代码）
5. 保存

**权限代码示例**:
- `script:*`: 剧本管理所有权限
- `account:*`: 账号管理所有权限
- `task:*`: 任务管理所有权限
- `permission:*`: 权限管理所有权限

### 4.3 高级功能

#### 4.3.1 智能分配

系统支持智能分配账号到不同节点和群组：
- 基于负载均衡
- 基于地理位置
- 基于账号状态
- 自定义分配策略

#### 4.3.2 监控和告警

**系统监控**:
- 实时性能指标
- API 响应时间
- 数据库查询性能
- 资源使用情况

**告警配置**:
1. 进入「设置」→「告警设置」
2. 创建告警规则
3. 设置告警条件（阈值、时间窗口等）
4. 配置通知渠道（邮件、Webhook、浏览器通知）
5. 保存规则

#### 4.3.3 数据导出

系统支持导出以下数据：
- 账号列表（Excel/CSV）
- 对话记录（JSON/CSV）
- 任务日志（JSON/CSV）
- 审计日志（JSON/CSV）

#### 7.2.3 Nginx 配置

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # WebSocket 支持
    location /api/v1/notifications/ws {
        proxy_pass http://127.0.0.1:8000/api/v1/notifications/ws;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 86400;
    }

    # API 路由
    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 前端路由
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
    }
}
```

## 常见问题

### 8.1 登录问题

**Q: 忘记密码怎么办？**
A: 联系管理员重置密码，或使用密码重置功能（如果已配置）。

**Q: 登录后提示权限不足？**
A: 检查用户角色和权限分配，确保有相应的权限。

### 8.2 账号管理问题

**Q: Session 文件导入失败？**
A: 
1. 检查 Session 文件是否有效
2. 确认文件格式正确
3. 检查账号是否已被其他系统使用

**Q: 账号无法启动？**
A:
1. 检查 Session 文件是否有效
2. 确认 Telegram API 凭证正确
3. 查看后端日志获取详细错误信息

### 8.3 任务执行问题

**Q: 自动化任务不执行？**
A:
1. 检查任务状态是否为「运行中」
2. 确认账号在线
3. 检查任务配置是否正确
4. 查看任务日志

**Q: 对话回复不正确？**
A:
1. 检查剧本配置
2. 确认角色分配正确
3. 查看对话历史分析问题

### 8.4 系统性能问题

**Q: 系统响应慢？**
A:
1. 检查数据库性能
2. 查看系统资源使用情况
3. 优化查询语句
4. 考虑增加服务器资源

**Q: WebSocket 连接失败？**
A:
1. 检查 Nginx 配置是否正确支持 WebSocket
2. 确认后端服务正常运行
3. 查看浏览器控制台错误信息

### 8.5 其他问题

**Q: 如何备份数据？**
A: 系统支持自动备份，也可以手动导出数据。数据库备份使用 PostgreSQL 的 pg_dump 工具。

**Q: 如何升级系统？**
A:
1. 备份数据库和配置文件
2. 拉取最新代码
3. 运行数据库迁移
4. 重启服务

### A. API 端点列表

主要 API 端点（完整列表请参考 API 文档）：

**认证**:
- `POST /api/v1/auth/login` - 登录
- `POST /api/v1/auth/logout` - 登出
- `POST /api/v1/auth/refresh` - 刷新 Token

**用户管理**:
- `GET /api/v1/users` - 获取用户列表
- `POST /api/v1/users` - 创建用户
- `PUT /api/v1/users/{id}` - 更新用户
- `DELETE /api/v1/users/{id}` - 删除用户

**剧本管理**:
- `GET /api/v1/group-ai/scripts` - 获取剧本列表
- `POST /api/v1/group-ai/scripts` - 创建剧本
- `PUT /api/v1/group-ai/scripts/{id}` - 更新剧本
- `DELETE /api/v1/group-ai/scripts/{id}` - 删除剧本

**账号管理**:
- `GET /api/v1/group-ai/accounts` - 获取账号列表
- `POST /api/v1/group-ai/accounts` - 创建账号
- `POST /api/v1/group-ai/accounts/{id}/start` - 启动账号
- `POST /api/v1/group-ai/accounts/{id}/stop` - 停止账号

**通知系统**:
- `GET /api/v1/notifications` - 获取通知列表
- `POST /api/v1/notifications/{id}/read` - 标记已读
- `WS /api/v1/notifications/ws/{user_email}` - WebSocket 推送
