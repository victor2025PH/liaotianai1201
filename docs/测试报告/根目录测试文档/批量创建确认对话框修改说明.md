# 批量创建确认对话框修改说明

## 问题
批量创建按钮弹出的是系统提示窗（`window.confirm`），而不是网页内的提示窗。

## 修改内容

### 1. 添加状态变量
在组件状态中添加了批量创建确认对话框的状态：
- `batchConfirmDialogOpen`: 控制对话框显示/隐藏
- `batchConfirmDialogTitle`: 对话框标题
- `batchConfirmDialogMessage`: 对话框消息
- `batchConfirmDialogAccountIds`: 要创建的账号ID列表
- `batchConfirmDialogScriptName`: 剧本名称
- `batchConfirmDialogResolve`: Promise resolve 函数，用于返回用户的选择

### 2. 修改 `handleBatchCreate` 函数
将原来的 `window.confirm` 替换为使用网页内的确认对话框：
- 设置对话框内容（标题、消息、账号列表、剧本名称）
- 使用 Promise 等待用户确认
- 用户点击"確定"时 resolve(true)
- 用户点击"取消"或关闭对话框时 resolve(false)

### 3. 添加确认对话框组件
在成功对话框后面添加了批量创建确认对话框：
- 使用 `AlertDialog` 组件（与现有的警告对话框一致）
- 显示要创建的账号列表（使用列表格式）
- 显示剧本名称
- 提供"取消"和"確定"按钮
- 支持通过点击外部或ESC键关闭（视为取消）

## 效果
- ✅ 使用网页内的对话框，而不是系统对话框
- ✅ 更好的用户体验，与整体UI风格一致
- ✅ 可以显示格式化的账号列表和剧本信息
- ✅ 支持滚动查看长列表

## 测试建议
1. 选择多个session文件
2. 点击"批量創建"按钮
3. 确认弹出的是网页内的对话框（不是系统对话框）
4. 检查对话框内容是否正确显示账号列表和剧本信息
5. 测试"取消"和"確定"按钮是否正常工作
